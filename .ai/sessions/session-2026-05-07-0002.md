## 이번 세션 작업

### 버그 수정 (`f6584d7`)

**증상**: 포털에서 워크스페이스 초기화 호출 시 아래 오류 반환.
```
[Errno 39] Directory not empty: PosixPath('/data/memory_store')
```

**원인**: `AX_PROJECT_ROOT=/data` 설정 시 workspace root = `/data`. reset 로직이
top-level 엔트리를 순회하면서 `memory_store/` 도 삭제 대상으로 판단. SQLite DB
파일(`ax.v2.db`)이 열려 있어 `shutil.rmtree` 실패.

**수정**: `coding_agent/web/artifacts.py` `_EXCLUDE_DIRS` 에 `"memory_store"` 추가.
에이전트 런타임 디렉토리 — 사용자 생성 파일이 아니므로 초기화 대상 외.

**변경 파일**:
- `coding_agent/web/artifacts.py` — `_EXCLUDE_DIRS` 에 `"memory_store"` 추가
- `tests/web/test_artifacts.py` — `test_workspace_reset_preserves_excluded_dirs` 에
  `memory_store` 생성 후 reset 후에도 유지되는지 검증 추가

**테스트**: `tests/web/` 14/14 passed

**포털 검증**: 재배포 후 워크스페이스 초기화 정상 동작 확인.

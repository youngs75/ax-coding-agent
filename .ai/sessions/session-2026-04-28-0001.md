## 중요 참고사항

### models.py / config.py 비대칭 (이번 세션의 진짜 교훈)
config 에 새 provider field 를 추가했는데 model 인스턴스화 path 에 분기를
안 추가하면 **silent fallback** 가능. anthropic / dashscope 같은 기존 provider
는 추가될 때 둘 다 함께 작업했는데 `litellm_portal` 만 누락된 것. 이게
*portal 환경에서만 노출되어* dev 환경에서 안 잡힌 회귀.

`config.py:148` 의 default 가 `"openrouter"` 인 것이 **silent failure** 의
공범. 모르는 provider 가 들어와도 KEY 없는 openrouter 로 fall back 해서 원인
멀어짐. `provider fail-fast` 후속 (위 §3) 이 방어선.

### portal serving pod ENV 화면 (확인 필요)
사용자가 ENV 화면 캡쳐 보여줬을 때 정확한 값으로 들어있었으나 (LLM_PROVIDER=
litellm_portal 등), 코드가 그것을 honor 하는 path 자체가 없었음. 따라서
이번 회귀의 *외관* 은 ENV 누락처럼 보였지만 *실체* 는 코드 버그. 향후
"포털 401 → ENV 누락 의심" 휴리스틱이 misleading 한 케이스로 기억할 것.

### apt-web → ax artifacts proxy 체인 (정상 확인됨)
`apt-web /coding/a2a/artifacts/{path}` proxy 가 정상 동작 (200 + valid
zip). `707fd24` + apt-web 의 `_proxy_artifacts` 가 모두 살아있음. e2e
시연 끝에 zip 다운로드 라운드트립 가능.

### chat UI "Error: no response" 의 정체
`chat_common.js:439` 에서 SSE 가 빈 stream 으로 닫혔을 때 띄우는 메시지.

yield 전에 예외" 또는 "ax 가 safe_stop / consecutive_errors 로 조기 종료"

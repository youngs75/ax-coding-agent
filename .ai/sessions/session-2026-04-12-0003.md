### 2. P2.5 — Execute 도구 hardening (런타임 hang 차단)
**동기**: 5차 round에서 `npm create vite@latest .` 인터랙티브 프롬프트로 인한 hang 4회 발생. LLM이 같은 명령을 무한 반복. 진단 결과 timeout이 LLM 인자(`timeout: int = 30`)로 노출되어 있어 LLM이 `600`초로 직접 늘림.

**구현 (`coding_agent/tools/shell.py` 완전 재작성)**:
- `timeout` 인자를 schema에서 **제거** → 환경변수 `EXECUTE_TIMEOUT` (기본 300s, max 600s, min 30s)
- `stdin=subprocess.DEVNULL` 모든 호출 강제
- 인터랙티브 명령 자동 보정: `apt-get install` → `-y`, `npm create/init`, `npx create-*` → `--yes`
- 보정 시 `[notice]` 프리픽스로 LLM에 학습 신호
- timeout 에러를 영어 + 진단 + **"Do NOT retry the same command — pick a different approach"** 가이드
- 위험 패턴 차단을 substring → regex로 정확화 (`rm -rf /workspace/build` 등 합법 명령 통과, `rm -rf /usr` 같은 진짜 위험 차단)

**검증**: 21 신규 테스트, 102/102 회귀 통과

### 3. P3 — LangGraph interrupt 기반 HITL (정공법)

**동기**: PRD가 사용자 요구사항 6개 → 36+ 항목으로 6배 인플레이션. 원인은 LLM이 모호한 요구사항을 마주쳤을 때 "PRD 템플릿"을 자동으로 채우는 본능. 해결책: planner가 PRD 작성 *전에* 사용자에게 핵심 결정을 묻기 (Claude Code의 `AskUserQuestion` 패턴).

**PoC 단계** (2건):
- 단일 그래프 interrupt + resume 동작 확인
- 중첩 그래프(outer → inner.ainvoke)에서 interrupt 전파 동작 확인
- ToolNode 안의 StructuredTool 함수에서 `interrupt()` 호출 가능 확인

**구현**:
- `coding_agent/core/loop.py`
  - orchestrator graph에 `InMemorySaver` checkpointer 추가
  - per-request thread_id (`uuid` 기반)
  - `AgentLoop.run`에 interrupt-resume 루프 + `ask_user` callback 인자
- `coding_agent/subagents/manager.py`
  - SubAgent graph도 `InMemorySaver` checkpointer 추가
  - `_paused_runs` 캐시 (key = `sha1(description+agent_type)[:16]`) — **재실행 시 SubAgent가 처음부터 다시 안 돈다**
  - `_invoke_graph` 헬퍼 분리 (fresh + resume 공통)
  - **idempotent 재호출**: `resume_value=None`이면 캐시된 payload 그대로 반환
- `coding_agent/subagents/models.py` — `SubAgentResult`에 `interrupt_payload`, `thread_id` 필드 추가
- `coding_agent/tools/task_tool.py`
  - 도구 함수가 `interrupt(payload)` 호출 → outer graph로 전파
  - 명시적 `except GraphInterrupt: raise` (broad except가 삼키지 않도록)
  - SubAgent 응답 받을 때까지 propagation 루프
- `coding_agent/tools/ask_tool.py` 신규
  - `AskUserQuestionInput` 스키마 (1–4 questions, header≤12자, options 2–4개, multi_select, allow_other)
  - 도구 함수가 `langgraph.types.interrupt(payload)` 호출
- `coding_agent/cli/question_renderer.py` 신규
  - Rich 기반 progress bar (`☐ Tech ☐ Mobile ✔ Submit`)
  - 번호 선택 / multi-select / `/skip` / free-form / 잘못된 입력 재시도
  - 입력 함수 주입 가능 (테스트용)
- `coding_agent/cli/app.py` — `_run_agent_streaming`에 interrupt-resume 루프 추가
- `coding_agent/subagents/factory.py` — planner role에 `ask_user_question` 추가, scope lock + 호출 가이드 명시
- `coding_agent/core/loop.py` — orchestrator SYSTEM_PROMPT의 PMS 도메인 누출 제거

## 발견된 회귀 5건 (P3.5 이월)

- **증상**: SPEC.md가 81줄, 우리 4섹션 헤더(`## 1. Goals` 등) 없음. LLM이 `write_file('SPEC.md')`로 우회

- **해결**: planner 프롬프트 강화 + `write_file`이 SPEC 경로 거부

### 2. verifier/reviewer/fixer 사이클 약화
- **증상**: 이번 E2E에서 verifier 1회(4초, 형식적), reviewer 0회, fixer 0회. 이전 6차는 각 2회씩
- **원인**: P3에서 orchestrator SYSTEM_PROMPT의 Phase 4를 "필요할 때만 호출. 단순 작업에는 생략 가능"으로 약화시킨 부작용
- **해결**: Phase 4 가이드를 "코드 변경 후 verifier 권장, 실패 시 fixer"로 다시 강화

### 3. 사용자 답변의 확장 해석 (모바일)
- **증상**: 사용자가 "Responsive web design only — Single codebase works on all devices" 답변. 그러나 산출물에는:
  - `AdminProjectRegistry-mobile.tsx`, `GanttChart-mobile.tsx`, `GanttChart.styled-mobile.ts` (별도 mobile 파일!)
  - `ResponsiveContainer.tsx`, `TouchTarget.tsx`, `mediaQueries.ts` (전용 모듈)
  - 4단계 breakpoint, `@media` 30개+
- **비교 대상 (Claude Code)**: `styles.css` 1개 + `@media (min-width: 768px)` 1개 + `grid-template-columns 1fr → 1fr 1fr`. 끝.
- **원인**: P3 scope lock이 planner에만 있고 coder에는 없음. 사용자 답변이 task description으로 전파 안 됨. SPEC가 부실해서 coder가 자유 해석
- **해결**: coder `_FORK_RULES`에 scope lock + interpret-narrow 룰, 사용자 답변을 task description에 자동 prepend

### 4. 자동 추가 컴포넌트

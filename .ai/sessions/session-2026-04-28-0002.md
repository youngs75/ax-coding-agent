### 2. 진단 — sse_emitter 의 두 회귀
**버그 1**: [sse_emitter.py:131-134](coding_agent/web/sse_emitter.py:131) 의
SubAgent-noise 억제 분기가 [L141, L152](coding_agent/web/sse_emitter.py:141)
의 `write_todos / update_todo → orchestrator.todo.change` 변환보다 *위* 에
있었음. planner 가 분해 후 write_todos 호출 시 `subagent_depth=1` 이라
즉시 `return None` → todo.change SSE 안 나감 → chat UI todo panel 비어있음.

**버그 2**: [stream_agent_events](coding_agent/web/sse_emitter.py:336)
의 payload 추출이 *legacy flat* `{question, choices, allow_free_text}` 만
읽음. 그런데 [_build_decomposition_interrupt_payload](coding_agent/core/loop.py:160)
가 만드는 실제 payload 는 *CLI 형태* `{questions: [{question, options:
[{label, description}], allow_other}]}`. `kind=="ask_user_question"` 검사는
통과하지만 `payload.get("question")` / `payload.get("choices")` 가 모두
비어있어 `input_required` SSE 가 빈 payload 로 흘러감 → modal 에 질문 없고
버튼도 없음.

→ 두 회귀가 같은 turn 에서 동시에 노출되어 chat UI freeze.

### 3. 정공법 fix (sse_emitter.py)

**Fix 1** — `write_todos` / `update_todo` 처리를 SubAgent-noise 억제 *위로*
끌어올림. SubAgent 안에서 호출돼도 todo.change 가 emit. 일반 tool 은 여전히
억제 (정책 보존). start 는 silent, end 의 todos 만 추출.

**Fix 2** — `_input_required_payload(payload, task_id)` private helper 추출.
CLI `questions[]` 형태 + legacy flat 형태 둘 다 cover:
- `questions[0].question` → top-level `question` (없으면 legacy `payload.get("question")` → default)
- `questions[0].options[].label/description` → `choices[].id/label/description` (없으면 legacy `payload.get("choices")`)
- `questions[0].allow_other` → `allow_free_text` (legacy 도)
- 단위 테스트 가능하도록 generator 본체 밖으로 분리.

### 4. 회귀 테스트 (`tests/web/test_sse_emitter.py` 신규, 10 케이스)
- planner 안 write_todos / update_todo → todo.change emit

- `allow_other` → `allow_free_text` 변환
- 빈 payload / 누락 question 필드 → default fallback

## 중요 참고사항

### sse_emitter 의 SubAgent-noise 억제 정책
- *현재 정책*: SubAgent 안의 모든 tool event 는 chat UI 에 안 보냄 (노이즈 절감).
- *예외* (이번 fix): `write_todos`, `update_todo` — ledger 갱신은 정보 가치 높음.
- 향후 SubAgent 안의 다른 *상태성* tool (예: `update_critic_verdict` 등 추가 시) 도 같은 패턴 — 억제 분기 위로 끌어올리는 흐름.

### `_build_decomposition_interrupt_payload` 의 형태
[loop.py:160-175](coding_agent/core/loop.py:160) — `kind="ask_user_question"`,
`questions[]`, `options[].label/description`. CLI 의 question_renderer 가
바로 받는 canonical 형태. web SSE 도 이 형태를 *기준* 으로 받아 변환.
**legacy flat 형태는 fallback** — 새 callers 가 추가될 때 questions[] 형태
사용 권장.

### chat UI freeze 의 정체 (이번 세션)
- chat_common.js 가 SSE event 들을 받긴 했지만:
  - `orchestrator.todo.change` 가 안 와서 todo panel 안 갱신
  - `input_required` 가 빈 payload 로 와서 modal 이 *물리적으로 떴어도* 질문/버튼 없어서 *효과적으로 freeze*
- 또는 modal 이 빈 상태로 안 보이게 처리됐을 수도 (chat_common.js:316 `features.hitlModal` 체크 외 추가 분기 미확인). 정확한 chat UI 상태는 사용자 시연에서 확인 가능.

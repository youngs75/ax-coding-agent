## v23 (minyoung-mah 0.2.0 승격) 후보

이번 세션에서 ax 에 구현된 패턴 중 *generic* 한 것 라이브러리화 검토:

### 승격 후보 A — Auto-verify chain 일반화

현 구현: `task_tool.py` 의 `_auto_verify_chain` — coder COMPLETED → 자동
verifier+fixer 사이클.

라이브러리 추상화: **mandatory transition edge** — consumer 가 role 전이
규칙 선언 (e.g., "coder→verifier→[fixer→verifier×2]→done"). 코딩 도메인은
coder→verifier, 다른 도메인은 다른 전이 (e.g., legal: drafter→reviewer).

라이브러리 새 API:
```python
orchestrator.set_post_completion_chain(
    "coder",
    chain=[("verifier", "pass"), ("fixer", "always_after_fail")],
    max_attempts=3,
)
```

### 승격 후보 B — Per-key invocation cap 일반화

현 구현: `task_tool.py` 의 `_FIXER_HARD_CAP` (`_fixer_attempts[task_id]`).

라이브러리 추상화: 임의 role + 임의 key extractor 로 cap. 코딩 도메인은
fixer + task_id, 다른 도메인은 다른 매핑. v22.4 의 #3 (task_id-free
확장) 도 이 추상화로 함께.

새 API:
```python
orchestrator.set_role_invocation_cap(
    role="fixer",
    cap=4,
    key_extractor=lambda desc: extract_task_id(desc) or "__gate__",
    on_cap_reached=lambda: "INCOMPLETE 메시지",
)
```

### 승격 후보 C — Update_todo evidence invariant (원래 v23 처방 #1)

`update_todo(completed)` 는 evidence (git diff 비어있지 않음 + task 의
expected file 존재) 와 교차검증 안 되면 reject. 모든 consumer 가 혜택.

라이브러리 위치: `minyoung-mah` 의 todo_tool 자체에 invariant.

### 승격 후보 D — Empty-diff session-end invariant (원래 v23 처방 #5)

orchestrator 종료 시도 시 hook: `git diff --name-only` 비어있으면 *완료
거부* + INCOMPLETE 시그널. v23 의 0 파일 + completed 류 차단.

라이브러리 위치: `minyoung-mah` 의 Orchestrator 종료 hook (Ralph Loop 강화).

### 승격 후보 E — Status enum 확장 (v22.4 #4 와 결합)

`completed` 외에 `verify_failed` (또는 `unverified`) 추가. ax 의 auto-advance
시점 조정 (#4) 과 함께 v23 라이브러리 변경.

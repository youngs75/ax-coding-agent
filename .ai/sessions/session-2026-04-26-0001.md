## 설계 의사결정

### 1) trim → autoCompact (라이브러리 승격)
`Context` 는 ax 5책임 정중앙. apt-legal 등 다른 consumer 가 동일 결함 재해결
필요 없게 `minyoung_mah.context` 로 추출. ax 외 소비자는 *opt-in* —
ContextManager 인스턴스화 안 하면 영향 0.

### 2) 도메인 비특화 prompt
Claude Code 의 `BASE_COMPACT_PROMPT` 직접 번역 (9 섹션). "Files and Code
Sections" 는 코딩/문서 작성/법률 컨설팅 모두 자연스럽게 적용. consumer 는
`prompt_override` 로 커스터마이즈도 가능.

### 3) Pair safety
`_adjust_tail_for_pair_safety` — tail 시작이 ToolMessage 면 짝 AIMessage 를
middle → tail 로 끌어와 strict pair 보장. compact 는 boundary 직전까지
요약하므로 tail 의 첫 메시지가 고아 ToolMessage 가 될 위험을 *원천 차단*.

### 4) Circuit breaker
`max_consecutive_failures=3`. 3회 연속 LLM summarize 실패 시 다음 호출은
`skipped:circuit_breaker` 로 즉시 반환 (원본 보존). 환경변수
`MINYOUNG_AUTO_COMPACT=0` 으로도 비활성 가능.

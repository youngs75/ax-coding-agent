### 3. Fix — ContextVar 기반 listener pattern (`6309253`)

**[`coding_agent/tools/task_tool.py`](coding_agent/tools/task_tool.py)**:

```python
SubAgentInvokeListener = Callable[[str, str, dict[str, Any]], None]

_subagent_invoke_listener: ContextVar[SubAgentInvokeListener | None] = (
    contextvars.ContextVar("...", default=None)
)

def set_subagent_invoke_listener(listener) -> Token:
    """Web stream 이 verifier/fixer invoke event 받기 위한 hook."""
    return _subagent_invoke_listener.set(listener)

def _emit_subagent_invoke(role, event_type, data):
    listener = _subagent_invoke_listener.get()
    if listener is None:
        return  # CLI 모드 — emit 비용 0
    try:
        listener(role, event_type, data)
    except Exception:
        log.warning("...", exc_info=True)  # auto-chain 작동 보호
```

`_auto_verify_chain` 의 verifier/fixer 호출 전후로 emit:
- start: `{description: <≤200char>, attempt: 1-based}`
- end: `{success: bool, elapsed_ms: int, attempt: int}`

PASS 판정 (`COMPLETED` 마커 + execute 실패 마커 부재) 을 emit 직전 분기와
중복 평가 — `invoke.end` 의 `success` 필드 정확성 보장. 비용 미미.

**[`coding_agent/web/sse_emitter.py`](coding_agent/web/sse_emitter.py)**:

```python
def _make_subagent_invoke_listener(extra_frames: deque) -> Callable:
    """auto-chain invoke → orchestrator.role.invoke.{start,end} SSE frame."""
    def _listener(role, event_type, data):
        if event_type == "start":
            desc = f"[auto-chain attempt {data['attempt']}] {data['description']}"
            extra_frames.append(sse(
                "orchestrator.role.invoke.start",
                {"role": role, "description": desc[:200]},
            ))
        elif event_type == "end":
            extra_frames.append(sse(
                "orchestrator.role.invoke.end",
                {"role": role, "success": ..., "elapsed_ms": ...},
            ))
    return _listener
```

`stream_agent_events` 가 stream 시작 시 listener 등록 (Token 으로 cleanup
보장), main loop 에서 매 LangGraph event 직전 extra_frames drain. try/finally
로 token reset — generator GC / 클라이언트 disconnect 시 contextvar leak 방지.

**`[auto-chain attempt N]` prefix** — chat UI 가 일반 SubAgent 호출과
auto-chain 호출을 시각적으로 구분 가능.

## 중요 참고사항

### ContextVar pattern 의 의미

`task_tool` (lib/coding-domain) 과 `sse_emitter` (web layer) 의 직접 import
의존을 만들지 않으면서 cross-layer 통신 가능. CLI 모드 / Docker headless 모드
는 listener 등록 안 함 → emit 호출 시 default `None` 반환으로 비용 0. portal
환경에서만 stream 별로 listener 등록 + 격리.

**향후 확장 가능성**: 다른 white-box 호출 (예: critic, sufficiency loop 의
auto-prompt) 도 같은 listener pattern 으로 surface 가능. wire-format 가시성을
위한 일반화된 hook.

### 결함 마스킹 누적의 종결 (4-layer)

| 0001 | apt-web chat_common.js parser | (단독) |
| 0002 | sse_emitter write_todos emit | 0001 마스킹 |
| 0003 | task_tool.auto_advance 우회 | 0002 마스킹 |
| 0004 | task_tool._auto_verify_chain 우회 | 0003 마스킹 |

→ **결함 A 가 결함 B 를 마스킹한 *네 번째* 사례**. 4-layer 누적이지만 각
layer 가 독립적 결함이라 *layer 별 단계적 검증* 으로 처리 가능. 단일 turn 에
모든 결함을 동시 진단했다면 코드 변경 폭이 커서 안전성 떨어졌을 것 — 사용자
e2e 재테스트마다 한 layer 씩 노출되는 패턴이 결과적으로 *작은 PR* 들로 분할.

### "안 쓰는 것" vs "안 보이는 것"

오늘 사용자 보고 — verifier/fixer 가 *안 쓰이는 것처럼 보임*. 실제로는:
- verifier 는 모든 task 에서 호출됨 (33개 테스트 = coder 5개 + verifier
  추가 검증 6개의 산물)
- fixer 는 첫 시도 PASS 라 호출 자체 없음 (정상 Decision D 흐름)

**원칙**: 사용자가 행동을 묻기 전에 *그 행동이 wire 에 보이는지* 항상 확인.
백엔드 로그 / 산출물 / wire response 를 교차 검증해야 정확한 진단 가능. UX
관점에서 **모든 SubAgent 협응이 보이는 것이 시스템 신뢰성의 핵심**.

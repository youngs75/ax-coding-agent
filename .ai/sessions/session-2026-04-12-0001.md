- manager.py: max_turns=50 + 텍스트 반복 감지 + 메시지 윈도우 트림 + 구조화된 출력

- 메시지 윈도우 head 보존 (System + HumanMessage 잘림 방지)

#### 3차 수정 (근본 원인 재발견)
- **SubAgent 메시지 트림 완전 제거** — SubAgent는 매번 Orchestrator로부터 신선한 컨텍스트를 받으므로 트림 불필요. 트림이 오히려 이전 작업을 잊게 만들어 같은 파일 반복 생성 유발
- **INCOMPLETE 시그널 추가** — max_turns 도달 시 Orchestrator에게 [Task INCOMPLETE] 전달
- **verifier 역할 추가** — 테스트/빌드 검증 전담 SubAgent (fast 모델, execute+read 권한, write 없음)

### 5. E2E 테스트 결과 (3차 빌드 진행 중)

### SubAgent 역할 체계 (6개)
| 역할 | 목적 | 도구 | 모델 티어 |
|------|------|------|----------|
| planner | PRD/SPEC 작성 | read, write, glob, grep | reasoning |
| coder | 코드 구현 | read, write, edit, execute, glob, grep | strong |
| reviewer | 코드 리뷰 | read, glob, grep | default |
| fixer | 버그 수정 | read, edit, execute, grep | strong |
| verifier | 테스트/빌드 검증 | read, execute, glob, grep | fast |
| researcher | 정보 조사 | read, glob, grep | default |

### 핵심 아키텍처 원칙 (이번 세션에서 확립)
1. **SubAgent에 메시지 트림 적용 금지** — 매번 신선한 컨텍스트로 시작, 128K 한도 대비 ~15K 사용
2. **Orchestrator에만 메시지 윈도우 적용** — 60K 토큰 문제는 여기서 발생
3. **Claude Code Fork Rules는 AX 구조에 부적합** — "텍스트 금지"는 자연스러운 턴 교대를 방해
4. **max_turns는 안전장치, INCOMPLETE 시그널은 필수** — Orchestrator가 미완성 작업을 인식해야 함
5. **Orchestrator의 역할: 작업 분할 + 신선한 컨텍스트 주입** — 작은 단위 위임이 핵심

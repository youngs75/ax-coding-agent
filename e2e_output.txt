4.1 간트 차트 API 데이터 포맷 불일치 (🔴 High Priority)

Spec 명세 (docs/SPEC.md p.119):


 [
   { "id": 1, "name": "요구사항 분석", "start": "2024-06-01", "end": "2024-06-15", "dependencies": [] }
 ]


실제 API 응답 (GanttService.findGanttData):


 return {
   project: projectDto,      // ❌ Spec 에 없는 project 객체 포함
   tasks: taskDtos,          // ✅ tasks 배열
   dependencies: dependencyDtos  // ✅ dependencies 배열
 }


UI 기대 포맷 (GanttChart.tsx):


 const [ganttData, setGanttData] = useState<GanttData[]>([]);
 // GanttData: { id, name, start, end, dependencies: string[] }


문제:

 1 API 가 { project, tasks, dependencies } 객체를 반환하지만, UI 는 GanttData[] 배열을 기대함
 2 frontend/src/api/gantt.ts 의 getGanttData 함수가 response.data 를 직접 반환하여 타입 불일치 발생
 3 dependencies 필드가 task-dependency 엔티티 기반인지, task 내 dependencies: string[] 인지 불명확

영향:

 • 🚨 런타임 오류 가능성: UI 가 data.map() 호출 시 data.project 에 대해 오류 발생
 • 🚨 의존성 라인 미표시: task.dependencies 배열이 채워지지 않아 SVG 의존성 선이 그려지지 않음

수정 제안 (Option A: API 응답 수정 - 권장):


 // backend/src/gantt/gantt.service.ts
 async findGanttData(projectId: string, user: MockUser) {
   // ... 기존 로직 (project, tasks, dependencies 조회)

   // tasks 에 dependencies 배열 포함하여 Spec 준수 포맷으로 반환
   const taskDtos = tasks.map((task) => {
     const taskDeps = dependencies
       .filter(dep => dep.taskId === task.id)
       .map(dep => dep.dependentTaskId);

     return {
       id: task.id,
       name: task.taskName,  // ✅ taskName → name 매핑
       start: task.startDate.toISOString().split('T')[0],
       end: task.endDate.toISOString().split('T')[0],
       dependencies: taskDeps,  // ✅ Spec 준수
     };
   });

   return taskDtos;  // ✅ 배열 직접 반환 (Spec 준수)
 }

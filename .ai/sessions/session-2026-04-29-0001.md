## 중요 참고사항

### 결함 A 와 B 의 직교성
- A (ALB timeout) 는 *연결 유지* 문제. keep-alive 로 *모든* 미래 e2e 에서
  자동 해소. fix B 측 매핑이 안 된 상태에서도 chat UI 가 freeze 만 안 하면
  modal 까지는 도달.
- B (event 매핑) 는 *내용* 문제. todo panel / role display 기능에 영향.
  진단 로그 받아서 정확한 fix.
- 두 결함이 같은 turn 에 동시 노출됐던 게 어제 0002 이후 의 freeze 의 정체.

### R-003 준수 (이번 사이클)
처방 후보 작성 시 *"LLM 측 형식 강제로 robustness 누락 메우기"* 패턴 영구
금지 (어제 0003 ratchet). 본 세션도 *"어떤 event 가 emit 되는지 추측해서
분기 추가"* 같은 추측 기반 fix 를 *후보로도 제시 안 함*. 진단 logging 으로
*실제 데이터* 본 후 정확한 매핑.

### keep-alive 패턴 (이번 세션)
asyncio.Queue + 백그라운드 drain task + `asyncio.wait_for(timeout=30)` 가
표준 패턴. graph.astream_events 의 generator 를 직접 race 하면 *cancel
시 graph 가 부정확하게 종료* 위험 → drain task 분리 + finally 에서 `cancel()
+ await` 으로 깔끔히 정리. HITL 대기는 `asyncio.shield(future)` 로 per-tick
timeout 이 future cancel 못 하게 보호 + deadline 직접 추적.

### apt-web 측 변경 불필요
apt-web 의 `_proxy_a2a_stream` 가 *raw chunk* 를 그대로 forward 하므로
`: keep-alive\n\n` SSE comment frame 도 자동 통과. chat_common.js 의 line
parser (`if (!line.startsWith('data: ')) continue;`) 가 SSE 주석을 무시.
양쪽 어디서도 추가 변경 없음.

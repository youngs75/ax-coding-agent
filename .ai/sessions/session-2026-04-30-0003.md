### 3. Fix — 매 event 마다 store snapshot diff (`74e3977`)

[`coding_agent/web/sse_emitter.py`](coding_agent/web/sse_emitter.py):

```python
def _todo_change_frame_if_changed(map_state, todo_store) -> bytes | None:
    """매 LangGraph event 직전 store snapshot 비교 — 변경 시 todo.change."""
    if todo_store is None:
        return None
    todos = _todos_from_store(todo_store)
    if todos is None or todos == map_state.get("last_todos_snapshot"):
        return None
    map_state["last_todos_snapshot"] = todos
    return sse("orchestrator.todo.change", {"todos": todos})
```

`stream_agent_events` 의 main loop 에서 LangGraph event 처리 직전 호출:

```python
diff_frame = _todo_change_frame_if_changed(map_state, todo_store)
if diff_frame:
    yield diff_frame
frame = _map_langgraph_event(kind, name, data, map_state)
if frame:
    yield frame
```

`write_todos` / `update_todo` 분기는 *snapshot 캐시 공유* — 중복 emit 방지:
```python
if todos == state.get("last_todos_snapshot"):
    return None  # store-diff path 가 이미 emit
state["last_todos_snapshot"] = todos
return sse("orchestrator.todo.change", {"todos": todos})
```

## 중요 참고사항

### `task_tool._auto_advance` 의 설계 의도

이 helper 는 SubAgent 위임 lifecycle (CREATED → ASSIGNED → RUNNING →
COMPLETED) 에 *맞춰* status 갱신을 자동화 — orchestrator LLM 이 매번
`update_todo` 호출하지 않아도 되도록 ergonomics 차원에서 도입. 다만 wire
관측성 측면에서는 *블랙박스* — LangGraph tool 이 아니라 직접 store 호출.

→ wire-format 동기화는 *별도 mechanism* 으로 보완해야 함. 본 fix 는 매 event
시점 store snapshot diff 검사로 보완. 향후 다른 직접-store-mutation 경로 추가
시 동일 패턴 따를 것.

### 매 event 검사의 성능 영향

`store.list_items()` 는 lock 잡고 list copy. N (task 수) 작아 비용 무시.
e2e 1회 ~100~500 events × O(N) = 무시. 변경 없으면 None 반환 + dict
comparison 만 — wire frame 추가 없음. 부담 zero.

### 0001 → 0002 → 0003 회귀 사이클 — 누적 결함의 단계적 노출

| 핸드오프 | 결함 | 마스킹 |

→ **결함 A 가 결함 B 를 마스킹한 *세 번째* 사례**. 진단 원칙 강화: layered

노출 가능. 본 세션은 0002 검증 후 즉시 새 결함 노출 → 빠른 진단 진행.

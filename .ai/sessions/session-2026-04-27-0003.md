## 중요 참고사항

### Harness 철학 — R-003 (오늘 신규)
처방 후보 작성 시 *"LLM 에 형식/행동 부담 추가"* 류 옵션은 후보로도 제시
금지. robustness 보강은 항상 harness 측 (정규식 완화, 정규화 layer, mechanical
fallback) 으로만. done-condition.md 같은 가이드는 *추천* 으로만 유지하고
prompt-level 강제 추가 금지.

### langfuse SDK 정합 (이번 세션)
이 repo 는 현재 `langfuse>=2.0,<3` 으로 핀. 따라서 호출 경로는 항상
`client.span()` / `client.event()` / `client.trace()`. v3+ 의
`start_observation` / `create_event` 는 **silently AttributeError** 가 됨
(observer 가 emit_failed debug 로그만 찍고 main flow 는 진행).

### TASK-NN 정규식 (이번 세션)
`\bTASK-\d+(?:\.\d+)?\b` — 1자리 + N.M sub-task 모두 흡수. planner 의
zero-pad 누락에 더 이상 의존하지 않음. 두 곳 (`task_tool.py`, `loop.py`)
이 같은 패턴을 공유하므로 한 곳 변경 시 다른 곳도 함께 변경할 것.

### Critic verdict 정규화 (이번 세션)
`_VERDICT_ALIASES`/`_TARGET_ALIASES` map 은 사소한 형식 변형 (대소문자,
동의어, dash/underscore) 을 흡수. 정말로 의미가 다른 verdict 만 escalate.

### Memory extractor (이번 세션)
JSON parse 실패 시 raw 안에서 첫 JSON array 패턴 추출 fallback. 머리말/
설명/펜스 모두 흡수. 그래도 못 찾으면 빈 list (이전과 동일한 soft failure).
